# AI Safety Daily Digest
**Date:** 2026-06-12  
**Theme:** AI Safety with emphasis on Agent Security, Agent Governance, Runtime Safety, Alignment, and Evaluation

## Highlights

当日研究进展在 Agent 运行时安全与多智能体协作风险领域取得显著突破。**SMSR: Certified Defence Against Runtime Memory Poisoning in Persistent LLM Agent Systems** 首次针对持久化记忆系统中的跨会话记忆投毒攻击提供了形式化防御证明，揭示了现有静态语料防御机制在面对动态记忆注入时的失效问题。**A Five-Plane Reference Architecture for Runtime Governance of Production AI Agents** 提出了生产级 Agent 运行时治理的五层参考架构，试图解决企业环境中数据边界溶解后的风险控制难题。此外，**MAStrike: Shapley-Guided Collusive Red-Teaming on Multi-Agent Systems** 引入了基于沙普利值的协同红队测试方法，量化了多智能体系统中角色间合谋攻击的系统性风险，为复杂工作流的安全评估提供了新视角。

## Agent 安全与治理

随着大语言模型向自主 Agent 演进，安全边界正从单一模型权重扩展至运行时环境、记忆存储及多智能体交互网络。当前最紧迫的风险在于持久化记忆系统的完整性，**SMSR: Certified Defence Against Runtime Memory Poisoning in Persistent LLM Agent Systems** 指出检索增强生成（RAG）Agent 在跨用户会话中积累的记忆构成了新的攻击面。攻击者无需触碰模型权重或代码，仅通过正常渠道注入精心构造的记忆，即可在未来会话中引导 Agent 行为，这种多会话记忆投毒（MSMP）攻击暴露了现有防御机制的盲区。作者声称其提出的认证防御方法能够抵御此类攻击，而现有的 RobustRAG 和 ReliabilityRAG 等静态语料防御方案因假设知识库固定而无法应对动态记忆污染。这一发现与 **Selection Integrity for LLM Graph Memory: An Accumulability Criterion for Information-Flow-Blind Retrieval** 的研究形成呼应，后者进一步指出基于图结构的长期记忆面临选择完整性危机，即未授权主体写入的结构可能改变已认证事实的选择路径，导致信息流控制失效。这表明单纯依赖记录来源验证不足以保障图记忆的安全性，需要引入全局选择步骤的审计机制。

在企业级部署层面，传统的数据边界防护已无法适应 Agent 的工作模式。**A Five-Plane Reference Architecture for Runtime Governance of Production AI Agents** 强调生产级 Agent 会读取上下文、调用工具并修改系统记录，风险内嵌于工作流序列之中。该架构主张将策略引擎扩展至单个许可动作的转换逻辑，而非仅仅监控数据进出边界。这与 **Exploring Systems-Thinking Approaches to Loss of Control Risk** 的观点一致，后者认为内部部署的 Agent 系统面临社会技术层面的失控风险，传统的模型级评估无法识别代码、基础设施或部署流程中的不可逆变更。因此，治理框架需要从单纯的合规检查转向对 Agent 行动后果的实时约束与审计能力。

在多智能体系统（MAS）中，安全风险呈现分布式特征。**MAStrike: Shapley-Guided Collusive Red-Teaming on Multi-Agent Systems** 针对分层多智能体系统提出了基于沙普利值的协同红队测试方法，旨在识别特定角色在权限提升和跨代理合谋中的关键贡献度。实验表明，现有的启发式目标选择方法难以覆盖复杂的协调攻击场景。这一结论得到了 **Smarter Saboteurs, Better Fixers: Scaling & Security in Linear Multi-Agent Workflows** 的支持，该研究发现模型规模扩大并不必然提升线性工作流的韧性，反而可能掩盖个体被劫持后的系统性崩溃风险。两者共同指向一个方向：多智能体安全不能仅关注单点防御，必须建立针对协作结构脆弱性的评估体系。同时，**The Emergence of Autonomous Penetration Capabilities in Large Language Model-Powered AI Systems** 警告称，具备自主渗透能力的 AI 系统正在成为现实，这要求我们在设计阶段就考虑防止 Agent 独立执行恶意操作的红线机制，而非事后补救。

## 工具/提示注入与运行时防御

提示注入与工具调用的安全性是运行时防御的核心战场。**PI-Hunter: Automated Red-Teaming for Exposing and Localizing Prompt Injections** 提出了一种自动化的代理审计框架，旨在主动暴露潜伏的提示注入漏洞及其传播路径。与现有主要优化攻击成功率的方法不同，PI-Hunter 侧重于开发者对潜在注入如何涌现的可见性，填补了红队测试在定位与归因方面的空白。然而，攻击视角的多样性同样重要，**Who Pays the Price? Stakeholder-Centric Prompt Injection Benchmarking for Real-world Web Agents** 批评了现有基准过于关注技术可行性而忽视危害分布的问题，主张以受害者为中心构建评估标准，因为实际风险高度依赖于受害者的身份与上下文。

视觉 - 语言 Agent 系统（VLAS）引入了新的后门威胁。**Beyond Attack Success Rate: Examining Trigger Leakage in Vision-Language Agentic Systems** 形式化了触发器精度失败的概念，指出当前的评估指标无法捕捉视觉后门是否仅在预期条件下触发隐藏行为。这意味着简单的攻击成功率指标可能掩盖了系统级的精确性缺陷。针对强化学习 Agent，**PolicyGuard: Towards Test-time and Step-level Adversary Defense for Reinforcement Learning Agent** 提出了测试时和步骤级的对抗防御，解决了现有后门防御要么依赖内部参数访问、要么局限于特定攻击类型的问题。这些工作共同表明，运行时防御需要从静态过滤转向动态的行为监控与干预。

隐私泄露在 Agent 轨迹中同样构成严重风险。**OCELOT: Inference-Leakage Budgets for Privacy-Preserving LLM Agents** 指出隐私不仅是单次输出的属性，而是整个轨迹的属性，累积性和双向性使得个人身份信息可能在信任边界间逐步泄露。虽然该工作涉及隐私保护，但其核心在于 Agent 运行时数据暴露的度量，这与 Agent 安全语境紧密相关。为了缓解此类风险，**CAPED: Context-Aware Privacy Exposure Defense for Mobile GUI Agents** 针对移动 GUI Agent 提出了上下文感知的隐私暴露防御，解决了截图观察导致的 incidental visual privacy exposure 问题，指出文本匿名化无法覆盖视觉推断线索。

## 模型对齐与可解释性

在确保 Agent 行为符合人类意图方面，推理过程的可靠性至关重要。**From Verdict to Process: Agentic Reinforcement Learning for Multi-Stage Fact Verification** 提出了 ProFact，一种基于代理强化学习的多阶段事实核查方法，旨在解决现有流水线优化孤立阶段导致的适应性协调不足问题。该方法通过强化学习优化各模块间的紧密耦合，提升了复杂声明处理的准确性。在辩论机制方面，**ARMOR-MAD: Adaptive Routing for Heterogeneous Multi-Agent Debate in Large Language Model Reasoning** 提出了一种无需训练的异构辩论框架，将辩论视为条件计算，通过预辩论协议路由和早期停止评估器来减少计算浪费和相关错误放大。

幻觉检测与可解释性是理解模型行为的基石。**Zero-source LLM Hallucination Detection with Human-like Criteria Probing** 提出了类似人类标准的探测机制，在无外部参考的情况下仅凭问答对进行幻觉检测，模拟了人类评估的多维度推理过程。然而，**Understanding the Rejection of Fixes Generated by Agentic Pull Requests -- Insights from the AIDev Dataset** 揭示了更深层的对齐问题，即 AI 生成的代码修复有接近一半被拒绝，反映了偏好访问与偏好合规之间的差距。这促使我们思考，如何在运行时强制执行用户修正，如 **Getting Better at Working With You: Compiling User Corrections into Runtime Enforcement for Coding Agents** 所提出的 TRACE 框架，该框架将用户修正编译为原子规则并在运行时强制执行，以弥补记忆系统的不足。

## 安全评估基准与开源工具

评估基准的演进直接反映了安全研究的重点转移。**Claw-SWE-Bench: A Benchmark for Evaluating OpenClaw-style Agent Harnesses on Coding Tasks** 引入了多语言 SWE-bench 风格基准，使异构 Agent 工具集能够在公平设置下进行比较，解决了通用 Agent 难以满足 Docker 工作空间契约的问题。针对搜索 Agent，**EvoBrowseComp: Benchmarking Search Agents on Evolving Knowledge** 提出了演化基准，利用实时网络遍历合成无污染的复杂问题，克服了静态知识基准易受测试集污染的影响。

开源工具生态正在为 Agent 安全提供工程化支持。**thu-nmrc/openloop** 是一个通用的循环工程框架，支持 Play-test-fix-verify-improve 周期，集成了日志、心跳、护栏和可审计的停止条件，为 Agent 的迭代开发提供了标准化环境。**cobusgreyling/loop-engineering** 则提供了实用的循环工程模式和参考，专注于设计提示和编排 AI 编码代理的系统。在记忆管理方面，**YMX899/MemoryCloud** 提供了一个类 GitHub 仓库式的 Agent 记忆云管理工具，允许在不同设备和会话间切换记忆状态，增强了 Agent 记忆的持久性与可控性。这些工具的出现标志着 Agent 安全正从理论研究走向工程落地，强调了运行时沙盒、工具调用审计和记忆治理的重要性。

## Looking Forward

尽管上述工作在 Agent 安全领域取得了重要进展，但仍存在若干未解决的理论问题与待验证假设。首先，关于持久化记忆的形式化认证，**SMSR** 虽提供了防御证明，但在高动态、多租户环境下的可扩展性与性能开销仍需大规模实证检验。其次，多智能体合谋风险的量化模型，如 **MAStrike** 采用的沙普利值方法，在超大规模异构 Agent 网络中的计算复杂度极高，如何平衡评估精度与效率是未来的挑战。最后，对于 Agent 失控风险的治理，**A Five-Plane Reference Architecture** 提出的五层架构尚缺乏工业界的广泛采纳与标准化，特别是在跨组织边界的 Agent 协作场景中，如何定义统一的运行时策略接口仍是开放性问题。未来研究需重点关注在资源受限环境下实现轻量级运行时监控，以及建立跨模态、跨平台的统一安全评估协议。
